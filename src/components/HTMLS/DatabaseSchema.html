Procurement Database Schema Workflow

Core tables and their relationships in the procurement management system

1
OrganizationTable
Multi-tenant foundation for the entire system. Each organization has its own procurement workflow.
Key Fields
name gstin address isActive
Relationships:
→ Users (1:N) → RFPs (1:N)
2
UserTable
Role-based users managing the procurement workflow with different permission levels.
Key Fields
role organizationId vendorId isActive
User Roles
PROCUREMENT_MANAGER FINANCE_TEAM VENDOR
Relationships:
← Organization (N:1) ← Vendor (N:1) → RFPs (1:N)
3
VendorTable
Vendor profiles with GST details and dealing keywords for automatic RFP matching.
Key Fields
companyName gstin dealingKeywords[] status
Status Values
PENDING_REVIEW APPROVED REJECTED
Relationships:
→ Users (1:N) → Quotations (1:N) → POs (1:N)
4
ProductCategoryTable
Hierarchical categorization of products like Laptops, Desktops, Network Equipment.
Key Fields
name description isActive
Relationships:
→ Products (1:N)
5
ProductTable
Specific products with flexible JSONB specifications and searchable keywords.
Key Fields
name categoryId specifications keywords[]
Relationships:
← ProductCategory (N:1)
6
RFPTable
Request for Proposals with line items, delivery requirements, and embedded conversation threads.
Key Fields
rfpNumber lineItems status conversationThreads
Status Flow
DRAFT PENDING_APPROVAL SENT_TO_VENDORS COMPLETED
Relationships:
← User (N:1) ← Organization (N:1) → Quotations (1:N)
7
QuotationTable
Vendor responses to RFPs with detailed pricing, line items, and supporting documents.
Key Fields
lineItemQuotes totalAmount status isShortlisted
Status Flow
OPEN SUBMITTED SHORTLISTED SELECTED
Relationships:
← RFP (N:1) ← Vendor (N:1) → POs (1:N)
8
PurchaseOrderTable
Generated purchase orders based on selected quotations with terms and delivery details.
Key Fields
poNumber lineItems totalAmount status
Status Flow
DRAFT SENT_TO_VENDOR ACKNOWLEDGED COMPLETED
Relationships:
← RFP (N:1) ← Quotation (N:1) ← Vendor (N:1) → GoodsReceipts (1:N)
9
GoodsReceiptTable
Final step tracking delivery confirmation, quality checks, invoice details, and vendor feedback.
Key Fields
receivedItems qualityCheck invoiceNumber vendorRating
Relationships:
← PurchaseOrder (N:1) ← User (N:1)
